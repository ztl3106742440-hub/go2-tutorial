> 本文是所有章节作者(含 Codex)必读的元规范。**写或改任何一章前,先通读本文**。样章请看 `docs/04-perception/11-slam-2d.md`。

## 0. 一句话原则

**面向零基础本科生,让他"抄作业也抄得明白"**。每一条命令、每一行代码、每一个参数,都要让读者能回答"这是什么""为什么要它""不加会怎样"。

---

## 1. 教材定位

| 项目 | 设定 |
|---|---|
| 读者画像 | 本科大学生,装好了 Ubuntu 22.04,Linux / Python / C++ 没有 0 基础但不精通,ROS2 可能只是听说过 |
| 正文主语言 | Python(主线) |

| 章节独立性 | 每章假设读者读过前面所有章,但不假设读过后面;跳章不保证能跑 |
| 正文禁用内容 | **任何绝对路径**(`/home/...` / `~/xxx/yyy/`) —— 路径仅出现在 Codex 交接文件中 |

## 3. 行文风格

### 3.1 语气与人称
- 用"我们"和"你",不用"笔者"/"用户"/"开发者"
- 讲动作用"现在/接下来/别急,先..."之类的口语化衔接,像老师在讲台上讲课
- 遇到难点要**预警**:*"这一步容易出错,原因是..."*
- 遇到非常规操作要**解释 why**:*"我们故意不用 xxx,因为它会..."*

### 3.2 技术词汇
- 中文术语 + 首次出现时括注英文:*"消息桥接(message bridge)"*
- ROS2/Ubuntu/Python 这类广泛使用的名词不翻译
- 命令和代码里的符号一律用反引号:`colcon build`

### 3.3 段落节奏
- 段落要短,一段 3–5 行,别写大块文字
- 讲完一个点空一行,别让读者喘不上气
- 长代码块之前用 1–2 句预告"接下来这段代码做三件事:..."
- 长代码块之后用 1–2 句点题"刚才这段,关键是..."

### 3.4 标点与排版
- 中文标点为主(,。;:""),代码/命令/英文保留英文标点
- 不用全角空格
- 避免"首先、其次、最后"的套话开头;直接讲事

## 4. 代码规范

### 4.1 代码块必须打语言标记

```python
# 对    ← python / bash / yaml / cpp / xml / json
```

```
# 错    ← 无语言标记,没语法高亮
```

### 4.2 Python import 必须逐包解释作用

```python
import rclpy                               # ROS2 Python 客户端库,节点的入口
from rclpy.node import Node                # 所有 ROS2 节点的基类,自定义节点继承它
from geometry_msgs.msg import Twist        # 标准速度消息,前后/左右/旋转三个分量
from unitree_api.msg import Request        # Go2 专用控制消息,需要 JSON 格式的参数
import json                                # 把 Python 字典转成 JSON 字符串
```

### 4.3 关键行要有注释,但禁止啰嗦

```python
# ✅ 解释 why
self.cmd_pub = self.create_publisher(Twist, "cmd_vel", 10)  # 10 是队列深度,落后了就丢旧消息

# ❌ 解释 what (代码已经说明)
self.cmd_pub = self.create_publisher(Twist, "cmd_vel", 10)  # 创建一个发布者
```

### 4.4 命令行代码块前必须有一句话说明

```bash
# 编译 go2_twist_bridge_py 包并加载到当前终端
cd ~/go2_tutorial_ws
colcon build --packages-select go2_twist_bridge_py
source install/setup.bash
```

### 4.5 长代码采用 MkDocs 代码注解

```python
twist = Twist()  # (1)!
twist.linear.x = 0.3  # (2)!
```

1. `Twist` 是 ROS2 标准速度消息,包含线速度(linear)和角速度(angular)
2. `linear.x` 单位是米/秒;Go2 EDU 安全上限 0.8 m/s

## 5. 原理铺垫规范

### 5.1 何时写【背景与原理】

| 章节类型 | 是否写 | 深度 |
|---|---|---|
| 纯软件/代码章(如键盘控制) | 可省略 | — |
| 带硬件章(雷达/IMU/摄像头) | **必写** | 硬件原理 + 选型对比 + 为什么选这个 |
| 带算法章(SLAM/Nav2/VLM) | **必写** | 算法思路 + 主要流派对比 + 本教材选用的方案 |
| 工程配置章(QoS/TF/launch) | 视情况写 | 只讲你要用的那部分概念 |

### 5.2 铺垫深度原则

**够用就好,别写成论文**。判断标准:读完这一节,读者能回答"我用的东西是什么""为什么用它"。不必推导公式,不必罗列所有分支。

示例(好):
> "激光雷达按扫描方式分三类:机械旋转式、MEMS 微振镜、固态相控阵。前两种在四足机器人上最常见。Go2 官方拓展坞支持两种型号 —— MID-360(固态全向)和 XT-16(机械旋转)。我们这本书用 MID-360,因为 ..."

示例(差):
> "激光雷达发明于 20 世纪 60 年代,其工作原理基于激光测距。激光测距分为 ToF、相位法、三角测量法 ... (继续写三百字,读者已经睡着)"

## 7. Admonition 使用约定

| 类型 | 语法 | 用途 |
|---|---|---|
| 提示 | `!!! tip "xxx"` | 小技巧、推荐做法 |
| 说明 | `!!! note "xxx"` | 补充信息,可跳过 |
| 警告 | `!!! warning "xxx"` | 常见坑、容易出错 |
| 危险 | `!!! danger "xxx"` | **实机安全相关**,必须读 |
| 折叠 | `??? note "xxx"` | 进阶/可选内容(C++ 附录、参数详解) |

## 9. 禁止事项

### 9.1 绝对禁止

- ❌ **教材正文出现任何绝对路径**(`/home/...` / `~/xxx/yyy`)
  - 工作空间用"你的工作空间"或 `~/go2_tutorial_ws/`(只作为教学示例统一前缀)
  - 代码文件位置用包的相对位置:*"在 `go2_twist_bridge_py/` 包的 `twist_bridge.py` 文件中"*
- ❌ **出现仅老大环境才有的信息**(IP/设备名/用户名等)
  - 唯一例外:Go2 官方默认 IP 段 `192.168.123.*`,这是全行业通用
- ❌ **写"笔者""我们团队""当时我"**之类的作者自叙
- ❌ **凭记忆写命令参数**。不确定时必须留 TODO 标记,不要瞎写

### 9.2 强烈不推荐

- ⚠️ 连续超过 80 行的代码块(拆成多段讲解)
- ⚠️ 未经任何铺垫就扔出一个参数表(先讲这个参数是什么、再列具体值)
- ⚠️ 用 emoji 装饰标题(✅/❌/⭐ 可在正文用,别进标题)
