# 命名约定

> 教材里的所有包名、节点名、文件名、变量名,都按本文走。**命名乱一次,读者就糊涂一次**。

---

## 1. 教材工程(本仓库)命名

### 1.1 目录
- 顶层目录中文:`go2实验指导书/`
- docs 内部分类用英文小写短名:`01-foundation/` `02-packages/` `04-perception/`
- 前缀 `NN-` 表示顺序,严格两位数字

### 1.2 Markdown 文件
- 数字前缀 + 英文短名 + `.md`,小写连字符分隔:`11-slam-2d.md`
- 前缀 = 该章在全书的章节号(不是在目录内的顺序)
- 不要中文文件名、不要空格

### 1.3 侧边栏/正文标题
- 在 `mkdocs.yml` 的 `nav` 里用中文,格式:**"第 N 章 章节名"**
- 正文 `# H1` 标题与 nav 保持一致

### 1.4 媒体资源
- 图片:`docs/assets/images/<章节编号>-<描述>.png`
  - 例:`11-slam-rviz-result.png` `13-nav2-costmap.jpg`
- 视频:`docs/assets/videos/<章节编号>-<描述>.mp4`

## 2. ROS2 工程(教材内示例代码)命名

### 2.1 工作空间
- 统一叫 `go2_tutorial_ws`
- 正文提及时写 `~/go2_tutorial_ws/`(学生在自己家目录创建)

### 2.2 ROS2 包前缀

| 用途 | 前缀 | 示例 |
|---|---|---|
| 接口定义(msg/srv/action) | `go2_tutorial_interfaces` | `go2_tutorial_interfaces/action/Nav` |
| 基础功能 | `go2_` | `go2_keyboard` `go2_twist_bridge` `go2_description` `go2_driver` |
| 感知 | `go2_` | `go2_sensors` `go2_slam` `go2_nav` |
| 交互应用 | `go2_` | `go2_voice` `go2_vision` `go2_missions` |
| Python 版单独包 | 原名 + `_py` | `go2_twist_bridge_py` `go2_keyboard_py` |

**规则**:
- 一律小写 + 下划线
- C++ 版与 Python 版同一功能时,Python 包加 `_py` 后缀
- 不用 `unitree_` 前缀(和官方包冲突)

### 2.3 节点名
- 节点名 = 主要功能,snake_case
- 不需要也不要加 `_node` 后缀(ROS2 风格)
- 示例:`twist_bridge` `keyboard_teleop` `pointcloud_timestamp_fix`

### 2.4 话题名
- **走 ROS2 标准**的话题(`/cmd_vel` `/odom` `/scan` `/map` `/tf`)保持原名
- **Go2 原生话题**(`/api/sport/request` `/utlidar/cloud_deskewed`)保持原名,不改
- **自定义话题**按功能小写 + 下划线:`/cmd_vel_teleop` `/utlidar/cloud_fixed`

### 2.5 Action / Service 名
- 名字 UpperCamelCase,动词或动名词
- 接口文件名:`Nav.action` `Cruising.srv` `SaveMap.srv`

### 2.6 Launch 文件
- 一个功能一个 launch,按用途命名:
  - `mapping.launch.py` —— 建图
  - `navigation.launch.py` —— 导航
  - `teleop.launch.py` —— 遥控
  - `display.launch.py` —— 只做可视化
  - `full_stack.launch.py` —— 全套一键启动
- 不要写 `go2_xxx.launch.py`(包名已经是 `go2_` 前缀,重复)

### 2.7 配置文件
- `<function>_params.yaml`:
  - `slam_toolbox_params.yaml`
  - `nav2_params.yaml`
  - `pointcloud_to_laserscan_params.yaml`
  - `driver_params.yaml`

## 3. Go2 坐标系命名

**Go2 用 `base` 不是 `base_link`**。这是踩过坑的地方,全教材统一:

| 坐标系 | 语义 |
|---|---|
| `map` | SLAM 发布的地图坐标系 |
| `odom` | Go2 驱动发布的里程计坐标系 |
| `base` | 机器人躯干中心(离地约 0.31m) |
| `imu` `radar` `utlidar_lidar` | 传感器坐标系 |

---

## 4. Python 代码命名

| 对象 | 风格 | 示例 |
|---|---|---|
| 类 | PascalCase | `TwistBridge` `KeyboardTeleop` |
| 函数/方法 | snake_case | `twist_callback` `publish_cmd` |
| 变量 | snake_case | `cmd_pub` `current_pose` |
| 常量 | UPPER_SNAKE_CASE | `MAX_LINEAR_VEL = 0.8` |
| 模块/文件 | snake_case | `twist_bridge.py` `sport_model.py` |
| API ID 常量 | 按官方命名 | `ROBOT_SPORT_API_ID_MOVE = 1008` |

---

## 5. C++ 代码命名

- 类:PascalCase(`TwistBridge`)
- 方法:snake_case(`publish_request`)
- 成员变量:snake_case + 末尾下划线(`request_pub_` `twist_sub_`)
- 宏/常量:UPPER_SNAKE_CASE

---

## 6. 禁止事项

- ❌ 不同章节对同一个东西用不同名字(比如 A 章叫 `go2_twist_bridge`,B 章叫 `twist_relay`)
- ❌ 包名混用大小写 / 驼峰
- ❌ 节点名加 `_node` 后缀
- ❌ launch 文件重复包前缀(`go2_twist_bridge/go2_twist_bridge.launch.py`)
- ❌ 教材里出现非 Go2 官方段的具体 IP / 具体用户名 / 具体机器名

---

## 7. 名称冲突时的决策

当本约定与原始素材(周记/整理版)不一致时:
1. **本约定优先**。原始素材只是参考,不是标准
2. Codex 发现冲突要**主动调整代码**,不要原样搬
3. 若命名约定本身遗漏了某类对象,记录到交接文件,由甘(Claude)补本文件
